## 🥇 方案1: 优化依赖结构（立即执行）

### 优化措施

#### 1.1 移除未使用的依赖

```bash
bun remove \
  @radix-ui/react-context-menu \
  @tanstack/react-virtual \
  date-fns \
  esbuild-wasm \
  isbinaryfile
```

**预期收益**: 减少 ~50MB node_modules

#### 1.2 添加缺失的依赖

```bash
bun add @clerk/clerk-js electron-log
```

**说明**: 这些依赖在代码中使用但未在 package.json 中声明

#### 1.3 优化 electron-builder 配置

**关键改进**:

```json
{
  "files": [
    "dist-electron/**/*",
    "dist-react/**/*",
    "package.json",
    "!node_modules/**/*",
    "node_modules/@anthropic-ai/claude-agent-sdk/**/*",
    "node_modules/better-sqlite3/**/*",
    "node_modules/electron-updater/**/*"
  ],
  "asarUnpack": [
    "node_modules/@anthropic-ai/claude-agent-sdk/**/*",
    "node_modules/better-sqlite3/**/*"
  ]
}
```

**效果**:
- 精确控制打包内容，排除不必要的 node_modules
- 扩展 asarUnpack，加速原生模块打包
- 预计减少 30-40% 打包时间

#### 1.4 执行优化

```bash
# 运行优化脚本
./scripts/optimize-deps.sh

# 测试打包效果
bun run dist:mac-arm64
```

### 预期效果

- **打包时间**: 减少 15-25%
- **安装包大小**: 减少 10-15%
- **风险**: 🟢 低
- **可回滚**: ✅ 是（已备份原配置）

## 🥈 方案2: 迁移到 electron-forge（中期）

### 为什么选择 electron-forge

1. **官方推荐**: Electron 官方主推的打包工具
2. **Vite 集成**: 与 Vite 深度集成，配置更简洁
3. **现代架构**: 基于插件系统，扩展性强
4. **长期收益**: 更好的维护和更新支持

### 迁移步骤

#### 2.1 安装依赖

```bash
bun add -D @electron-forge/cli @electron-forge/maker-dmg @electron-forge/maker-zip @electron-forge/plugin-vite
```

#### 2.2 创建配置文件

```javascript
// forge.config.js
module.exports = {
  packagerConfig: {
    asar: true,
    icon: './cherry2-square.png',
    protocols: [
      { name: 'Cherry Agent Auth', schemes: ['cherry-agent'] }
    ],
    extraResource: [
      'resources/preset-skills'
    ]
  },
  makers: [
    {
      name: '@electron-forge/maker-dmg',
      config: {
        format: 'ULFO'
      }
    },
    {
      name: '@electron-forge/maker-zip',
      platforms: ['darwin']
    }
  ],
  plugins: [
    {
      name: '@electron-forge/plugin-vite',
      config: {
        build: [
          {
            entry: 'src/electron/main.ts',
            config: 'vite.main.config.ts'
          }
        ],
        renderer: [
          {
            name: 'main_window',
            config: 'vite.config.ts'
          }
        ]
      }
    }
  ]
}
```

#### 2.3 更新 package.json

```json
{
  "scripts": {
    "package": "electron-forge package",
    "make": "electron-forge make"
  }
}
```

### 预期效果

- **打包时间**: 减少 15-25%
- **配置复杂度**: 降低 30%
- **维护成本**: 降低（官方支持）
- **风险**: 🟡 中等（需要测试）

## 🥉 方案3: 混合方案（仅当 Bun 是瓶颈时）

### 实施方案

#### 3.1 保留 Bun 用于开发

```json
// package.json
{
  "scripts": {
    "dev": "bun run dev:react & bun run dev:electron",
    "test": "bun test"
  }
}
```

#### 3.2 使用 pnpm 用于打包

```json
{
  "scripts": {
    "dist:mac-arm64": "pnpm install --frozen-lockfile && pnpm run transpile:electron && pnpm run build && electron-builder --mac --arm64"
  }
}
```

#### 3.3 同步 lock 文件

```bash
# 创建同步脚本
cat > scripts/sync-lockfiles.sh << 'EOF'
#!/bin/bash
# 从 bun.lock 生成 pnpm-lock.yaml
bun install
pnpm import
EOF

chmod +x scripts/sync-lockfiles.sh
```

### 注意事项

⚠️ **高维护成本**:
- 需要维护两套 lock 文件
- 依赖版本可能不一致
- CI/CD 需要安装两个包管理器
- 团队协作复杂度增加

### 预期效果

- **打包时间**: 减少 20-30%（仅当 Bun 兼容性是瓶颈）
- **维护成本**: 增加 50%
- **风险**: 🔴 高

### 建议

**先测试 pnpm 打包速度**:

```bash
# 测试 pnpm 打包
pnpm install
pnpm run dist:mac-arm64

# 对比 Bun 打包
bun install
bun run dist:mac-arm64
```

**仅当 pnpm 明显更快时才考虑混合方案**

## 🏅 方案4: Docker 打包（可选）

### 适用场景

- CI/CD 环境
- 需要环境一致性
- 多平台并行打包（Linux/Windows）

### 实施方案

```dockerfile
# Dockerfile.build
FROM node:20-alpine

# 安装 pnpm
RUN npm install -g pnpm

WORKDIR /app

# 复制依赖文件
COPY package.json pnpm-lock.yaml ./

# 安装依赖（利用 Docker 缓存）
RUN pnpm install --frozen-lockfile

# 复制源代码
COPY . .

# 构建
RUN pnpm run transpile:electron && pnpm run build

CMD ["pnpm", "run", "electron-builder"]
```

### 限制

⚠️ **macOS 打包限制**:
- macOS 打包必须在 macOS 上运行
- Docker 无法解决 macOS 代码签名问题
- 只适用于 Linux/Windows 打包

### 预期效果

- **打包时间**: 减少 5-15%（主要是缓存）
- **环境一致性**: 提升
- **风险**: 🟡 中等
